Executive Summary

The 'Quin House (thequinhouse.com) faces CRITICAL member data exposure risk through multiple independent vectors. The PeopleVine CMS platform exposes a fully documented Swagger REST API (113 endpoints) with CORS `*`, no rate limiting on auth endpoints, and an email enumeration vulnerability.

A public GitHub repository (sbrand-claudecode/quin-calendar) contains working authentication code with a live Stripe publishable key, automated refresh token rotation every 6 hours, and patterns enabling full member directory data extraction.

Combined with DMARC `p=none` (anyone can spoof @thequinhouse.com emails) and publicly accessible Azure blob storage, the club's member PII is exposed across six distinct attack paths. An attacker with moderate skill could enumerate all member email addresses, spoof club communications, phish credentials, and extract the full member database through the PeopleVine API.

Detailed Findings

PII-01: GitHub Code Leak — Live Stripe Key + Auth Code

CRITICAL Data Exposure
🔗 github.com/sbrand-claudecode/quin-calendar

The public GitHub repository contains scrape.js — a working script that logs into members.thequinhouse.com, extracts the PeopleVine session token from localStorage, and calls PeopleVine APIs to fetch events and attendee lists.

NamesEmailsEvent Data
Impact: Stripe LIVE publishable key pk_live_1VlvJMD6gCzrYwwLNKJiVshM exposed. GitHub Actions auto-rotates PeopleVine refresh tokens every 6h using secrets QUIN_REFRESH_TOKEN, QUIN_EMAIL, QUIN_PASSWORD. Full PII extractable. GITHUB_TOKEN has contents:write permission.
Remediation: Make repo private immediately. Rotate Stripe key. Audit sbrand-claudecode org for other repos. Check Actions cache artifacts. Rotate PeopleVine credentials.

PII-02: PeopleVine API Full Surface (113 Endpoints)

CRITICAL Data Exposure
🔗 rest.peoplevine.com/swagger/v1/swagger.json

Complete PeopleVine Swagger 2.0 spec publicly documents 113 API endpoints. Key endpoints: GET /api/directory/members (full member list), GET /api/customers (addresses, notes, payments), GET /api/events/{id}/attendees, GET /api/account/personal-details.

NamesEmailsPhonesAddressesPaymentsEventsPhotos
Impact: Any valid PeopleVine auth token can extract full member database: names, emails, phones, addresses, membership status, payment history, event attendance, personal details, customer notes, profile photos.
Remediation: PeopleVine platform issue — Quin must escalate. Implement API gateway validation. Restrict Swagger doc to authenticated users.

PII-03: CORS `*` on All API Responses

CRITICAL Auth Bypass

Every PeopleVine API response includes Access-Control-Allow-Origin: *. Any website visited by a logged-in Quin member can make authenticated cross-origin API calls.

HTTP/1.1 200 OK
access-control-allow-origin: *
access-control-allow-credentials: true
access-control-allow-methods: GET, POST, PUT, PATCH, DELETE, OPTIONS
Impact: Cross-origin data exfiltration. Combined with DMARC phishing (PII-07), attacker sends spoofed email → victim clicks → malicious page extracts PeopleVine data.
Remediation: Restrict CORS to members.thequinhouse.com, portal.thequinhouse.com, apply.thequinhouse.com. Never use wildcard with credentials.

PII-04: Email Enumeration via /api/account/registered

CRITICAL PII Leakage

GET /api/public/account/registered/:username returns different responses for registered vs. unregistered emails — enabling complete member email list enumeration.

Member Emails
# Registered: {"registered": true, ...}
# Not registered: {"registered": false}
Impact: Complete member email list for targeted phishing. Combined with DMARC spoofing, attacker can send perfectly spoofed club emails to every member.
Remediation: Remove endpoint or require auth. Return uniform responses. Rate-limit. Add Cloudflare WAF rule.

PII-05: SSO Bootstrap Public Endpoint

CRITICAL Auth Bypass

GET /api/public/token/sso/bootstrap?code=XYZ accepts an OAuth authorization code and exchanges it for an SSO session token. Valid codes → token without credentials.

Impact: Any attacker who obtains an OAuth auth code can get a PeopleVine session token. Codes obtainable via phishing, OAuth misconfiguration, or Envoy redirect interception (PII-09). No rate limiting detected.
Remediation: Require client authentication. Rate limit. Shorten code expiry. IP-based restrictions. Audit Auth0 redirect URI whitelist.

PII-06: Azure Blob Storage Public Read

CRITICAL Data Exposure

peoplevine.blob.core.windows.net/files/1356/ publicly accessible with anonymous read permissions.

PhotosDocuments
Impact: Member-uploaded identification documents, contract PDFs, event photos publicly accessible. Company ID 1356 = The 'Quin House.
Remediation: Restrict ACLs to private. Audit current files. Use SAS tokens. Enable Azure Storage logging.

PII-07: DMARC `p=none` — Email Spoofing

CRITICAL PII Leakage

DMARC: v=DMARC1; p=none — anyone can spoof @thequinhouse.com emails.

DMARC: v=DMARC1; p=none;
SPF:   v=spf1 mx include:_spf.peoplevine.net include:spf.protection.outlook.com -all
Impact: Classic phishing — "Your membership renewal required" from @thequinhouse.com. CEO fraud. Combined with email enumeration (PII-04), every member is targetable.
Remediation: Set p=quarantine; pct=100 immediately. After 30 days: p=reject; pct=100. Configure DKIM for all senders.

PII-08: No Rate Limiting on Token Endpoints

HIGH Auth Bypass

Password grant has 5-attempt lockout, but SSO bootstrap, refresh token, and email enumeration endpoints have no detected rate limiting.

Impact: Brute force OAuth codes, mass email enumeration, unlimited refresh token rotation, organizational credential stuffing.
Remediation: IP-based rate limiting at PeopleVine/Envoy. Rate-limit SSO bootstrap. Cloudflare WAF rules.

PII-09: Envoy Auth Proxies Leak api_tokens in Redirect

HIGH Data Exposure

Istio Envoy proxies at loginpe.thequinhouse.com and loginse.thequinhouse.com leak api_tokens in redirect URL during OAuth flow.

Impact: Phishing can steal tokens from URL. Referrer leakage to external resources. Tokens in browser history. Shared computer risk.
Remediation: POST-based token delivery. PKCE. Clear tokens from URL. Referrer-Policy: no-referrer. Short token TTL.

PII-10: PeopleVine Portal Session/IP Leak

HIGH PII Leakage

apply.thequinhouse.com HTML source exposes session IDs, visitor IPs, Company No (1356), Domain No (12013), Page Style No (2295).

Impact: Visitor IPs collected and potentially exposed. Internal IDs useful for API enumeration. Session IDs enable hijacking.
Remediation: Remove debug data from production HTML. Strip __VIEWSTATE internal data. Remove IP hidden fields. Enable customErrors.
